# 2026-05-23 Coding Test

## 문제 1

- 사이트: Programmers
- 문제명: x만큼 간격이 있는 n개의 숫자
- 난이도: Lv. 1

### 처음 접근

```text
x씩 증가하는 값이므로 인덱스와 동시에 조회 가능
for문을 사용하여 0부터 차례로 접근하면서
i번 자리에 x * (i + 1) 저장하기
```

### 내 풀이

```java
long[] answer = {};
answer = new long[n];
for(int i = 0; i < n; i++){
  answer[i] = (long)x * (i + 1);
}
return answer;
```

### 다른 풀이

```java
return LongStream.iterate(x, i->i+x).limit(n).toArray();
```

### 배운점

- `x * (i + 1)` 부분을 실행할 때 int의 범위를 넘어설 수 있으므로 먼저 long 타입으로 강제 형 변환 후 코드를 진행 시켜야한다.
- 배열 타입이 `long[]`이어도 계산 과정이 `int`끼리 이루어지면 오버플로우가 발생할 수 있다.

### 다시 볼 부분

- 타입별 max값 확인하기, LongStream의 iterate 메서드 찾아보기

## 문제 2

- 사이트: Programmers
- 문제명: 나머지가 1이 되는 수 찾기
- 난이도: Lv. 1

### 처음 접근

```text
n을 x로 나눈 나머지가 1이되는 가장 작은 자연수 x 찾기
n % x == 1
x는 2부터 1씩 증가하면서 조회
  => `% 0` 의 경우 0으로 나눌 수 없어서 오류 발생
  => `% 1` 의 경우 항상 0이기 때문에 수행 할 필요 없음
```

### 내 풀이

```java
int answer = 0;
for(int i = 1; i < n; i++){
  if( n % i == 1){
    answer = i;
    break;
  }
}
return answer;
```

### 다른 풀이

```java
return IntStream.range(2, n).filter(i -> n % i == 1).findFirst().orElse(0);
```

### 배운점

- `% 0` 의 경우 런타임 오류 발생
- `% 1` 의 경우 항상 0이기 때문에 수행할 필요 없음

### 다시 볼 부분

- `% 0`과 `/ 0`은 런타임 오류
- `% 0.0` 과 `/ 0.0`은 NaN과 Infinity

## 문제 3

- 사이트: Programmers
- 문제명: 최댓값과 최솟값
- 난이도: Lv. 2

### 처음 접근

```text
문자열에 공백으로 구분된 숫자 => split(" "); => 리턴값은 문자열 배열
이 중 최소와 최대 구하기
문자열 "(minNum)(maxNum)" 리턴하기
```

### 내 풀이

```java
String answer = "";
int maxNum = Integer.MIN_VALUE;
int minNum = Integer.MAX_VALUE;
String[] stringNum = s.split(" ");

for(String st : stringNum){
  int selectedNum = Integer.parseInt(st);
  maxNum =  selectedNum > maxNum ? selectedNum : maxNum;
  minNum = selectedNum < minNum ? selectedNum : minNum;
}
answer += minNum + " " + maxNum;
return answer;
```

### 다른 풀이

```java
String[] tmp = str.split(" ");
int min, max, n;
min = max = Integer.parseInt(tmp[0]);
for (int i = 1; i < tmp.length; i++) {
    n = Integer.parseInt(tmp[i]);
  if(min > n) min = n;
  if(max < n) max = n;
}

return min + " " + max;
```

### 배운점

- 변수 명 명확하게 잘 지어서 구분 편하게 하기

### 다시 볼 부분

- 굳이 int의 최대 최소를 넣을 필요 없이 0번 인덱스의 값을 사용해도 된다.
